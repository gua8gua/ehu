## 一、项目概述

Ehu 是一个基于 C++ 的游戏引擎雏形，采用分层架构与事件驱动设计，核心层与平台/渲染实现解耦。当前仅支持 Windows，默认使用 OpenGL + GLFW + ImGui 作为后端。

---

## 二、已完成模块

### 2.1 核心层（Core）

- **Application**：单例应用，持有 Window、LayerStack、ImGuiLayer，主循环负责清屏、层更新、ImGui 渲染与窗口刷新
- **Layer / LayerStack**：层栈管理，支持普通层与 Overlay，事件从栈顶向栈底派发
- **Log**：基于 spdlog 的日志系统，宏 `EHU_CORE_`* / `EHU_*`
- **Core.h**：`EHU_API`、`EHU_ASSERT`、`BIT(x)` 等通用宏
- **KeyCodes / MouseCodes**：键鼠码定义

### 2.2 事件系统（Events）

- **Event**：基类与 `EventDispatcher` 模板
- **KeyEvent**：KeyPressed、KeyReleased、KeyTyped
- **MouseEvent**：MouseMoved、MouseScrolled、MouseButtonPressed、MouseButtonReleased
- **ApplicationEvent**：WindowClose、WindowResize

### 2.3 平台抽象（Platform）

- **GraphicsBackend**：图形后端枚举，支持环境变量 `EHU_GRAPHICS_BACKEND` 选择
- **Window**：窗口抽象接口与 `Window::Create()` 工厂
- **Input**：输入抽象与 `Input::Init()` 工厂
- **RendererAPI**：渲染 API 抽象（`SetClearColor`、`Clear`）与 `RendererAPI::Init()` 工厂

### 2.4 ImGui 集成（ImGui）

- **ImGuiLayer**：集成到层栈，提供 `OnImGuiRender` 回调
- **ImGuiBackend**：抽象接口与 `Create(backend)` 工厂

### 2.5 后端实现（Backends/OpenGL_GLFW）

- **WindowsWindow**：基于 GLFW 的窗口实现，事件回调转 Ehu::Event
- **WindowsInput**：基于 GLFW 的键盘/鼠标输入
- **ImGuiBackendGLFWOpenGL**：ImGui 与 GLFW + OpenGL 的绑定
- **OpenGLRendererAPI**：清屏实现（`glClearColor` / `glClear`）

### 2.6 示例与构建

- **SandBox**：示例可执行程序，实现 `CreateApplication()`，包含 `ExampleLayer`
- **构建**：支持 Visual Studio 生成器与 Ninja，Ninja 可生成 `compile_commands.json` 供 IDE 使用
- **开发环境**：已配置 clangd（`.clangd`）与 VS Code/Cursor 的 C++ 扩展说明

### 2.7 第三方依赖（vendor）

- GLFW、GLAD、imgui、glm、spdlog 已集成
