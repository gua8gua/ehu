## 三、仪表盘模块

### 3.1 数据层（DashboardStats）

- **位置**：`Ehu/src/Ehu/ImGui/DashboardStats.h`
- **内容**：单例 `DashboardStats::Get()`，包含：
  - **Timing**：FrameTimeMs、CpuTimeMs、GpuTimeMs（-1 表示 N/A）、UpdateMs、PhysicsMs、RenderSubmitMs、ScriptingMs；以及 120 帧环形历史（FrameTimeHistory、GpuTimeHistory、HeapMbHistory）。
  - **Rendering**：DrawCalls2D/3D、Triangles2D/3D、Vertices2D/3D（近似）、TextureBindings、ShaderSwitches（每帧 SnapshotRenderingCounters 写入后清零本帧计数）。
  - **Memory**：VramBytes、HeapBytes、ActiveEntities。
- **写入方**：Application::Run（Timer 分段、RenderQueue 统计、GetActivatedScenes 实体数、MemoryStats/GetVramBytes）；OpenGL 层（Bind 时 AddTextureBinding/AddShaderSwitch）；FlushAll 前后 BeginGpuTiming/EndGpuTiming，下一帧 GetLastGpuTimeMs。

### 3.2 可视化层（DashboardLayer）

- **位置**：`Ehu/src/Ehu/ImGui/DashboardLayer.h`、`DashboardLayer.cpp`
- **行为**：只读 DashboardStats，在 ImGui 中分块展示 Timing（含 CPU/GPU 进度条、子系统表格、Frame Time/GPU Time 折线）、Rendering（表格）、Memory & Resources（表格 + Heap 折线）；Frame Time/FPS 按绿/黄/红着色。
- **显示控制**：`ImGuiWindowVisibility::ShowDashboard`；F3 切换；菜单 Window → Dashboard 勾选；再次打开时 `GetDefaultWindowPos("Dashboard", ...)` + `SetNextWindowPos(..., ImGuiCond_Always)` 回到默认位置。

### 3.3 窗口可见性与默认位置

- **ImGuiWindowVisibility**（`ImGuiWindowVisibility.h` / `.cpp`）：共享 `ShowStats`、`ShowDashboard`、`ShowImGuiDemo`；`GetDefaultWindowPos(windowName, outX, outY)` 基于主视口 WorkPos 与菜单栏高度，Stats 左上、Dashboard 其右。
- **DebugLayer**：仅当 `ShowStats` 为真时绘制；再次打开时设默认位置；窗口带关闭按钮，与菜单勾选同步。
- **ImGuiLayer**：顶部 `BeginMainMenuBar()`，Window 菜单下 `MenuItem("Stats", ..., &ShowStats)`、`MenuItem("Dashboard", "F3", &ShowDashboard)`、`MenuItem("ImGui Demo", ..., &ShowImGuiDemo)`；继续保留 `SnapFloatingWindowsBackToViewport()` 将完全移出视口的窗口拉回边缘。

## 四、平台与后端扩展

- **MemoryStats**（`Platform/MemoryStats.h` / `.cpp`）：`GetProcessHeapBytes()`，Windows 下使用 `GetProcessMemoryInfo`（WorkingSetSize），非 Windows 返回 0。
- **RendererAPI**：虚方法 `BeginGpuTiming()`、`EndGpuTiming()`、`GetLastGpuTimeMs()`（默认 no-op 返回 -1）、`GetVramBytes()`（默认 0）；OpenGL 实现 GL_TIME_ELAPSED 查询与 NV `GL_NVX_gpu_memory_info`（可用时）写 VramBytes。

## 五、构建与脚本约定（写入 rules）

- **默认配置与编译**：使用 **scripts 文件夹下脚本** 进行配置与编译，与 docs/README.md、开发日志 #002 一致：
  - **配置**：`.\scripts\gen_compile_commands.ps1`（在 VS 开发者环境中生成 build-ninja 与 compile_commands.json）。
  - **编译**：`.\scripts\build_ninja.ps1`（Ninja 构建 EhuLib + SandBox）。
