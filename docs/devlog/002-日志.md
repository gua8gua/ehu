# 开发日志 #002 — 渲染框架开发

**日期**：2025-03-04

---

## 一、本阶段概要

本阶段完成 Ehu 的**整套渲染框架**开发：在「Render API 层（API/平台相关）」与「Renderer 层（平台无关）」二分架构下，实现从几何提交到 Present 的完整管线，并配套 3D 示例与 Ninja 构建脚本，使引擎具备可用的 2D/简单 3D 绘制能力。

---

## 二、渲染架构设计

### 2.1 分层与职责

渲染系统采用**两层结构**，与具体图形 API 解耦：

```
┌─────────────────────────────────────────────────────────────────┐
│  Renderer 层（平台无关）                                            │
│  Camera、Renderer2D、Scene、Material 等                             │
│  不依赖 OpenGL/DirectX 等，只通过 Platform 抽象提交绘制               │
└─────────────────────────────────────────────────────────────────┘
                                    │
                                    │ 提交绘制命令 / 设置 uniform
                                    ▼
┌─────────────────────────────────────────────────────────────────┐
│  Render API 层（API/平台相关）                                       │
│  RendererAPI、VertexBuffer、Shader、Texture、Framebuffer、Pipeline 等 │
│  接口在 Platform/，实现在 Backends/OpenGL_GLFW/                      │
└─────────────────────────────────────────────────────────────────┘
```

- **Render API 层**：提供缓冲、着色器、纹理、帧缓冲、状态、渲染通道、管线等底层抽象，由当前选定的 GraphicsBackend（目前为 OpenGL + GLFW）实现。
- **Renderer 层**：提供相机、场景图、2D/3D 渲染器、材质等高层接口，只依赖 Platform 的 Render API 接口与 Core/Events，便于后续扩展多后端或替换实现。

### 2.2 目录与约定

- **Platform/**：渲染相关接口（RendererAPI、RenderContext、Pipeline、VertexBuffer、IndexBuffer、VertexArray、Shader、Texture2D、Framebuffer、Window、Input 等）。
- **Backends/OpenGL_GLFW/**：上述接口的 OpenGL 实现（OpenGL* 前缀），采用 **OpenGL 3.3 兼容 API**，避免依赖 4.5 专有接口。
- **Renderer/**：平台无关的渲染逻辑（Camera、Renderer2D、Renderer3D 占位、Scene、Material 等），对外统一通过 **RendererModule.h** 或按需包含子目录头文件。

详见 [RENDERER_ARCHITECTURE.md](../RENDERER_ARCHITECTURE.md)。

## 三、Render API 层实现

### 3.1 核心抽象

| 模块 | 职责 | 说明 |
|------|------|------|
| **RendererAPI** | 清屏、绘制、状态、渲染通道 | SetClearColor、Clear(flags)、DrawIndexed、SetDepthTest/SetBlend/SetCullFace/SetViewport、BeginRenderPass/EndRenderPass；Init/Shutdown |
| **RenderContext** | 渲染上下文与 Present | Init/Shutdown、GetAPI()、SetCurrentWindow、SwapBuffers；Application 通过其初始化与交换缓冲 |
| **VertexBuffer / IndexBuffer** | 顶点与索引数据 | Create、Bind、布局/数量；OpenGL 实现 |
| **VertexArray** | VB/IB 绑定与顶点属性 | AddVertexBuffer、SetIndexBuffer、Bind；OpenGL 实现 |
| **Shader** | 着色器与 uniform | Create(源码)、Bind、SetMat4/SetFloat4/SetInt 等；OpenGL 实现 |
| **Texture2D** | 2D 纹理 | Create(width,height,data)、CreateFromFile（需 EHU_USE_STB_IMAGE）、Bind(slot)、SetData；OpenGL 实现 |
| **Framebuffer** | 离屏渲染与 MRT | FramebufferSpec、Create、Bind/Unbind、Resize、GetColorAttachmentRendererID；支持深度/模板；OpenGL 实现 |
| **Pipeline** | 管线状态与一次提交 | SetShader/SetVertexArray/SetDepthTest/SetBlend/SetCullFace、Bind、DrawIndexed；Platform 抽象 + OpenGL 实现 |

### 3.2 主循环中的使用

- Application 在每帧开始时通过 **RenderContext::GetAPI()** 调用 **SetClearColor**、**Clear(ClearColor \| ClearDepth)**，再遍历 Layer 执行 OnUpdate。
- 各 Layer 内可调用 Renderer2D::BeginScene / DrawQuad / EndScene；Renderer2D 内部绑定 Shader、设置 u_ViewProjection、提交四边形时通过 **RendererAPI::DrawIndexed** 完成绘制。
- 帧末由 **Window::OnUpdate()**（或 RenderContext::SwapBuffers）完成事件轮询与交换缓冲。

## 四、Renderer 层实现

### 4.1 相机

- **OrthographicCamera**：2D 正射投影，位置、旋转、RecalculateViewMatrix，与 glm 集成。
- **PerspectiveCamera**：3D 透视投影（fov、aspect、near/far），位置与欧拉角，用于简单 3D 场景。

### 4.2 Renderer2D

- **BeginScene(OrthographicCamera)** / **BeginScene(const Camera&)**：写入 ViewProjection、绑定 Shader、上传 u_ViewProjection；后者支持任意相机（含 PerspectiveCamera），用于 3D 透视。
- **DrawQuad(position, size, color)**：支持 vec2/vec3 位置，内部构造 transform、设置 u_Color、绑定四边形 VAO、调用 RendererAPI::DrawIndexed。
- **EndScene()**：当前为立即模式，无额外批处理逻辑。
- 四边形几何（顶点 + 索引）与默认 Shader 在 **Renderer2D::Init()** 中创建，与 Platform 的 VertexBuffer、IndexBuffer、VertexArray、Shader 对接。

### 4.3 场景与材质

- **Scene / SceneNode**：层次化场景图，节点变换传播（占位/基础实现）。
- **Material**：材质属性与 Bind/Unbind 接口，Bind 尚未与 Shader 属性对接，为后续扩展预留。
- **Renderer3D**：接口占位（BeginScene(PerspectiveCamera)、Submit），待前向渲染等实现。

### 4.4 入口与初始化

- **Renderer::Init()** / **Renderer::Shutdown()**：统一初始化/关闭 Renderer2D、Renderer3D 等，在 Application 构造/析构中、窗口与 RenderContext 就绪后调用。
- 对外头文件：**RendererModule.h** 统一包含 Renderer、Camera、Renderer2D、Renderer3D、Scene、Material。

## 五、3D 示例与脚本

### 5.1 SandBox 3D 示例

- 新增 **Example3DLayer**：使用 **PerspectiveCamera**(45°, 1280/720, 0.1, 100)，相机位于 (0, 0, 6)。
- 每帧开启深度测试后，通过 **Renderer2D::BeginScene(m_Camera)**（传入透视相机）与多次 **DrawQuad** 绘制 5 个不同 z 的彩色四边形（前排 z=0 红/绿，中排 z=-2 蓝/黄，后排 z=-3.5 紫），用于验证透视与深度遮挡。
- 主循环 Clear 使用 **ClearColor \| ClearDepth**，保证 3D 深度缓冲正确。

### 5.2 Ninja 构建脚本（scripts/）

- **gen_compile_commands.ps1**：在已加载 VS 环境的同一进程中执行 cmake（通过 **vcvarsall.bat**），生成 build-ninja 与 compile_commands.json，避免 Launch-VsDevShell 启动新壳导致后续命令无 cl/标准库路径。
- **build_ninja.ps1**：同样通过 vcvarsall 注入环境后执行 ninja，保证 EhuLib + SandBox 可完整编译。
- 使用方式：`.\scripts\gen_compile_commands.ps1` 配置，`.\scripts\build_ninja.ps1` 编译。

## 六、其他说明

### 6.1 CMake 与头文件

- Debug 下仅对非 MSVC 添加 `-g`，避免 MSVC 报“未知选项 -g”（Ehu 与 SandBox 均已调整）。
- **RenderContext.h** 包含 **RendererAPI.h**，保证调用 GetAPI() 时类型完整；**RendererAPI.h** 使用 **Core/Core.h**，保证 SandBox 等 include 路径下可解析。

### 6.2 当前构建与运行

- **配置**：`.\scripts\gen_compile_commands.ps1` 生成 build-ninja 与 compile_commands.json。
- **编译**：`.\scripts\build_ninja.ps1` 完成 EhuLib + SandBox 的 Ninja 构建。
- **运行**：`build-ninja\bin\Debug\SandBox.exe` 运行后可见透视下的多层彩色四边形与深度遮挡。
